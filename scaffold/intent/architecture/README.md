# Architecture Intent

Use this folder for durable system concepts, boundaries, design principles, governance foundations, and cross-cutting constraints.

Create workstreams under:

- `intent/architecture/workstreams/<yyyy-mm-dd-topic>/`

Typical artifacts:

- conceptual model notes
- service or module boundaries
- state ownership boundaries
- data lifecycle and governance posture
- security or compliance foundations
- architecture decision inputs

Do not place detailed endpoint contracts here unless they define a broader architecture boundary. Put request/response details in `intent/api-design/`.
