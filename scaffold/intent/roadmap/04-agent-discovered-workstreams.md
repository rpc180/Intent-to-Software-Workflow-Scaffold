# Agent-Discovered Workstreams

This tracker captures workstreams discovered by agent-team runs. It is separate from `intent/roadmap/03-feature-backlog.md` so discovery status, next artifacts, and recommended agent teams can be tracked without changing implementation backlog priority.

`03-feature-backlog.md` remains the canonical feature backlog. This file should link to backlog IDs when they exist and should not create a second source of delivery priority.

For faster agent runs, start with `intent/00-active-context.md` and the relevant workstream `CURRENT.md` before reading this entire tracker or any long historical artifact.

## Tracking Rules

- Add one row per discovered workstream.
- Keep suggested intent location aligned with `intent/intent-taxonomy.md`.
- Keep a concise `CURRENT.md` in active or recurring workstream folders.
- Use `Related backlog IDs` to connect to canonical backlog rows.
- Use `Priority posture` to guide next planning, not to override backlog priority.
- Move implementation-ready scope into the feature backlog or a slice plan before editing runtime code.
