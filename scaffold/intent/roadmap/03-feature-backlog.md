# Feature Backlog

Use this file to capture newly discovered features before implementation.

Purpose:

- preserve intent for ideas discovered during build/test/demo
- keep feature requests tied to project goals and user value
- enable prioritization without losing context

For faster agent runs, start with `intent/00-active-context.md` and targeted `rg` lookups for specific `F-###` rows before reading this full backlog.

## Intake Rules

- Add one row per feature idea.
- Include user value and intent alignment notes.
- Assign a proposed slice, milestone, or `Unscheduled`.
- Update status as work progresses.
- Keep this file as the canonical priority source for feature work.

## Status Values

- `Discovered`
- `Planned`
- `Ready for slice planning`
- `Ready for implementation`
- `Implemented (MVP)`
- `Accepted / closed`
- `Parked`
